2.1 MODELO DE COMUNICACIÓN OSI

Modelo OSI, ¿qué es?

El modelo OSI es el modelo de la interconexión de sistemas abiertos, y sus siglas provienen del inglés Model Open System Interconection. Este es un modelo de referencia para los protocolos de red, este estándar fue desarrollado en 1980 por una federación global de organizaciones que representan aproximadamente a 160 países.


Esta normativa está formada por 7 capas que define las diferentes fases por las que deben pasar los datos para viajar entre dispositivos sobre una red de comunicaciones. En los cuales los usaremos tanto en la parte de sistemas como para la parte de redes.

El modelo OSI no es la definición de un tecnología ni un modelo de red en sí mismo, lo que hace es definir la funcionalidad de ellos, para conseguir un estándar. Desde este modelo se han creado numerosos esquemas de protocolo de red.

Capas del Modelo OSI

· Aplicación
· Presentación
· Sesión
· Transporte
· Red
· Enlace
· Físico

Modelo de aplicación de la séptima capa

Esta capa ofrece la posibilidad a las aplicaciones de que puedan acceder a las demás capas del modelo, y define los protocolos de intercambio de datos. En esta capa se encuentran aplicaciones de correos, gestores de base de datos o servidores de ficheros.

Esta capa también permite que los usuarios puedan realizar acciones y ejecutar sus instrucciones, a través de una interfaz gráfica o una interfaz de línea de comandos.

Un ejemplo que podemos tener para una capa de aplicación seria el protocolo SMTP, que es usada para emails.

Modelo de la sexta capa o capa de presentación

Esta capa es la que se encarga de ver y representar los datos para poder mostrarlos al usuario, en esta capa se analiza las representaciones que pueda tener tanto un ordenador, como otros dispositivos que puedan conectarse y así poder traducir el idioma del contenido, semántica y sintaxis de los datos transmitidos ya que pueden tener diferentes formas de manejarla. En esta capa también se permite que los datos sean cifrados y comprimidos.

Esta sexta capa también se encarga de trabajar el contenido que el usuario o el técnico quiere ver.

Modelo de la capa de sesión

Esta capa se encarga de mantener y controlar el enlace que se establece entre dos ordenadores que están transmitiendo datos, y se ocupa de reanudar las comunicaciones en caso de haber alguna interrupción.

Esta capa en la gran parte de casos es imprescindible o parcialmente imprescindible para el uso de servicios. También se ocupa del mapeo de las direcciones que transportara la información y trabajara con las capas inferiores, que podemos encontrar en el Modelo OSI.

Modelo de la capa de transporte

Esta capa es la que permite el transporte de los datos, entre las máquinas de origen y destino, ya que es independiente el tipo de red que se usa, para distribuir los datos, estos se encuentran dentro del paquete que se envía. Estos paquetes, dependiendo del segmento o diagrama, pueden corresponderse con los paquetes TCP o UDP.

La diferencia entre el paquete TCP y UDP es que el TCP es el paquete de transmisión que está orientado a la conexión y el paquete UDP es un paquete que se usa sin conexión y pueden perderse paquetes por el camino.


En esta capa también es donde se utilizan los puertos lógicos que usamos por ejemplo para navegar por internet por el modelo http ya que va al puerto 80 o con el protocolo seguro que va al puerto 443, en la cual se proporcionará una calidad suficiente para que cuando se transmita el paquete llegue correctamente entre el host de destino y el host de origen que hace la petición.

Modelo de la capa de red

La capa de red es la encargada de la identificación del enrutamiento entre dos o más redes que están conectadas. En este nivel lo que hará es que los datos puedan llegar de una máquina a otra. Siendo capaz de realizar los saltos y encaminamientos necesarios para poder recibir el mensaje.

El protocolo IP es el más conocido y es el que se encarga de esto, hay más tipos de protocolos, tales como IPX, APPLETALK o ISO 9542.

Modelo de enlace de datos

La capa de enlace es la que se encarga del direccionamiento físico, el acceso al medio y la detección de los errores. Este es uno de los aspectos importantes a la hora de conectar dos máquinas. Ya que forma parte esencial para la creación de sus protocolos básicos también ya que está entre las capas física y de red.

Esta capa lo que hace es construir los bits de información y controlar con otros elementos que se transmita de forma correcta.

Modelo físico

Esta capa es la más baja y se encarga tanto de la topología de la red, como de la conexión global del equipo hacia la red. Se encarga también de los elementos físicos y los procedimientos a nivel electrónico, lo cual hace que dé estabilidad a que el bit viaje de un punto a otro sin alteraciones. Este modelo garantiza también la conexión, aunque no garantiza la fiabilidad.

Un ejemplo de estos medios de transmisión físicos son los cables de pares trenzados, o las ondas para la wifi. Tenemos algunas de las normas ISO que tendríamos que cumplir en esta capa, por ejemplo, el modelo ISO 2110.
